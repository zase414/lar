\section{Popis řešeného problému}
\label{sec:popis}

Robot \textit{Ferenc} má za úkol autonomně opustit garáž, lokalizovat míček
v prostoru, objíždět jej po předdefinované trajektorii a poté se vrátit
do výchozí pozice. Celý systém je implementován na platformě Turtlebot využitím RGB kamery a 3D hloubkového senzoru (point cloud).

\textcolor{red}{\textbf{[POZNÁMKA:]} Zde je vhodné doplnit i informaci o \textbf{hardwarové konfiguraci} (např. zda jde o Turtlebot 2 nebo 3 a typ hloubkové kamery – RealSense/Kinect), což specifikace vyžaduje.}

\subsubsection{Detekce míčku}

Pro lokalizaci míčku je využita maska v barevném prostoru HSV, která je méně náchylná na změny intenzity osvětlení než prostor RGB. Cílem je izolovat pixely odpovídající žlutozelené barvě míčku.

Algoritmus pracuje v následujících krocích:
\begin{itemize}
    \item \textbf{Prahování:} Obraz je transformován do binární masky na základě limitů $H \in [34, 65]$, $S \ge 50$ a $V \ge 40$.
    \item \textbf{Morfologická filtrace:} Pro odstranění šumu (izolovaných pixelů) je aplikována operace \textit{opening} s jádrem $5\times5$.
    \item \textbf{Geometrická filtrace:} V masce jsou nalezeny kontury. Aby byl objekt považován za míček, musí jeho plocha $A(C)$ překročit 120 px a kruhovitost $\kappa(C)$ musí být větší než 0,7.
\end{itemize}

Kruhovitost je definována jako:
\begin{equation}
  \kappa(C) = \frac{4\pi \cdot A(C)}{P(C)^2},
  \label{eq:circularity}
\end{equation}
kde $P(C)$ je obvod kontury. Pokud objekt podmínky splňuje, je jeho pozice v obraze určena jako střed nejmenší popsané kružnice.

\subsubsection{Detekce brány (nevyužito)}

Původní návrh počítal s detekcí výjezdu z garáže (brány) pomocí dvou vertikálních modrých značek. Algoritmus byl navržen tak, aby identifikoval dva nejvíce vlevo umístěné objekty splňující specifické proporce.

Klíčové parametry detekce:
\begin{itemize}
    \item \textbf{Barva:} Prahování v HSV pro modrou barvu ($H \in [110, 140]$, $S \ge 50$, $V \ge 40$).
    \item \textbf{Tvar:} Detekované kontury musely mít plochu větší než 300 px a poměr výšky k šířce (aspect ratio) minimálně 1,5, což odpovídá vertikálně orientovanému obdélníku.
    \item \textbf{Logika středu:} Po seřazení kandidátů podle souřadnice $x$ byl vypočten aritmetický střed mezi dvěma nejbližšími značkami, který definoval střed brány pro průjezd.
\end{itemize}

\subsubsection{Odhad vzdálenosti}

Vzdálenost k míčku je odhadována ze středních hodnot hloubkového senzoru
v okolí středu detekovaného konturu:

\begin{equation}
  d = \frac{1}{|W|} \sum_{(u,v)\in W} z(u,v),
  \label{eq:depth_estimate}
\end{equation}

kde $W$ je okno velikosti $(2r+1) \times (2r+1)$ se středem v $(u_0, v_0)$
a $r \in \{1, 2\}$ závisí na poloměru detekovaného míčku v pixelech.

\subsubsection{PID regulátor otáčení}

Úhlová odchylka $e_\theta(t)$ je regulována PID regulátorem:

\begin{equation}
  u(t) = K_p\, e_\theta(t)
         + K_i \int_0^t e_\theta(\tau)\,\mathrm{d}\tau
         + K_d \frac{\mathrm{d}e_\theta}{\mathrm{d}t}.
  \label{eq:pid}
\end{equation}

Použité konstanty jsou $K_p = 2{,}1$, $K_i = 0{,}067$, $K_d = 0{,}18$.

\subsubsection{Hexagonální trajektorie}

Střed míčku v souřadnicích odometrie je označen jako $\mathbf{b} = (b_x, b_y)$.
$i$-tý uzel hexagonu (pro $i = 0, 1, \ldots, 5$) leží na kružnici
o poloměru $r_h = d_{\text{stop}} + r_{\text{míček}}$:

\begin{equation}
  \mathbf{w}_i = \mathbf{b} +
    r_h \begin{pmatrix} \cos\!\left(\dfrac{i\pi}{3}\right) \\[4pt]
                        \sin\!\left(\dfrac{i\pi}{3}\right) \end{pmatrix}.
  \label{eq:hexagon}
\end{equation}

\section{Změny specifikace v průběhu řešení}
\label{sec:zmeny}

V průběhu implementace a testování došlo k následujícím změnám oproti původnímu konceptu:

\begin{itemize}
  \item \textbf{Zájezd do garáže:} Přestože byl vyvinut funkční modul pro detekci fialových obdélníkových značek pomocí barevné segmentace a analýzy poměru stran, v konečné logice nebyl využit. Hlavním důvodem bylo zjištění, že pro úspěšný zájezd do garáže postačuje zapamatování odometrie. Detekce brány by byla spolehlivější, avšak musela by se vyvinout metoda jak dostat robota do osy brány garáže, aby robot nenarazil při zájezdu do bočních stěn.
  \item \textbf{Úprava morfologie:} Původní jednoduché prahování bylo doplněno o morfologickou operaci opening, která se ukázala jako nezbytná pro stabilitu detekce míčku v prostředí s drobnými žlutými artefakty dřevotřískových stěn.
\end{itemize}

\subsection{Struktura projektu}

Projekt je rozdělen do tří modulů:

\begin{description}
  \item[\texttt{ferenc.py}] Hlavní řídicí logika robota – třída \texttt{Ferenc}.
  \item[\texttt{visuals.py}] Detekce míčku a obdélníkových značek.
  \item[\texttt{image\_processing.py}] Zpracování hloubkového obrazu a detekce volného prostoru.
\end{description}

\textcolor{red}{\textbf{[NEDOSTATEK:]} V této sekci chybí \textbf{popis konečného automatu (FSM)} nebo celkového toku řízení. LAR vyžaduje popsat, jak se robot přepíná mezi stavy (např. vyjíždění z garáže -> hledání -> objezd).}

\subsection{Způsob spuštění}

Robot se spustí příkazem:

\begin{lstlisting}[language=bash, caption={Spuštění robota.}]

\end{lstlisting}

Vyžaduje prostředí \texttt{robolab\_turtlebot}, OpenCV a NumPy.

\section{Příloha A – Úplný zdrojový kód}
\label{app:kod}

\begin{lstlisting}[language=Python, caption={Výpočet průměrné hloubky (zkráceno).}]
def get_depth(turtle, center_x, center_y, radius):
    if radius < 2:
        return None
    pc = turtle.get_point_cloud()
    r = 1 if radius < 16 else 2
    depths = [
        float(pc[center_y + i][center_x + j][2])
        for i in range(-r, r + 1)
        for j in range(-r, r + 1)
        if pc[center_y + i][center_x + j][2] is not None
        and float(pc[center_y + i][center_x + j][2]) > 0.1
    ]
    return sum(depths) / len(depths) if depths else None
\end{lstlisting}
